# Hackathon submission checklist

## Complete in the repository

- [x] Project uses Codex and GPT-5.6 and meets the required-tool requirement.
- [x] Category selected: **Apps for Your Life**.
- [x] Devpost-ready feature and functionality description is prepared.
- [x] Public repository is accessible and includes an MIT license.
- [x] README explains where Codex accelerated implementation and which product,
  engineering, and design decisions remained with the team.
- [x] README explains GPT-5.6 specialist agents, synthesis, and deterministic scoring.
- [x] Credential-free fixture instructions let judges test without API keys or accounts.
- [x] `.env`, OAuth tokens, private data, mailbox exports, and generated personal images
  are ignored and absent from tracked files and history.
- [x] `make check` passes locally.
- [x] Plugin/Dev Tool installation clause is not applicable: RealCart is entered as an app.

## Must be completed in Devpost

- [ ] Push this reviewed branch and merge it into `main` before submitting the repository URL.
- [ ] Record a clear demo with audio and keep it under three minutes.
- [ ] Show the product, Codex collaboration, and GPT-5.6 multi-agent implementation.
- [ ] Use no copyrighted music, logos, personal receipts, email addresses, or secret screens.
- [ ] Upload the video publicly to YouTube and paste its URL into Devpost.
- [ ] Run `/feedback` in the core Codex project thread and paste the Session ID into Devpost.

## Optional

- [ ] Add a hosted fixture demo URL if time allows. This is helpful but is not required for
  the Apps for Your Life category because the repository includes a credential-free judge path.
